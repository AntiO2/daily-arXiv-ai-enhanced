### [1] [Case Studies and Reflections on Agentic Software Engineering for Rapid Development of Digital Music Instruments](https://arxiv.org/abs/2605.14016)
*Matthew John Yee-King*

Main category: cs.SE

TL;DR: 本文探讨了将智能体软件工程（ASE）应用于音频软件开发，通过三个案例（重新实现Music Mouse、移植Continuator、开发3D tracker界面）展示了ASE在降低门槛、提升互操作性和软件 longevity 方面的潜力。


<details>
  <summary>Details</summary>
Motivation: 数字音乐乐器开发面临寿命短、互操作性差和入门门槛高三大挑战，作者希望借助ASE技术降低开发门槛，使非程序员音乐家也能参与创作，同时提升软件的可维护性和互操作性。

Method: 采用案例研究方法，使用ASE技术在C++/JUCE框架下开发三个音频软件项目：1) 重实现Laurie Spiegel的Music Mouse；2) 将Pachet的Continuator从Python移植；3) 为tracker音序器开发3D界面（OpenGL）。通过自民族志方法分析提示日志和开发快照，总结有效实践。

Result: 成功完成三个案例的ASE辅助开发，记录了人类开发者与ASE协作的经验，识别出在该领域使用ASE的有效实践模式。

Conclusion: ASE在音频软件开发中具有降低门槛、促进创新的潜力，未来需与非程序员音乐家合作评估该方法的效果。

Abstract: The article explores the use of agentic software engineering (ASE) in the development of innovative audio software. It begins with a review of background work that lays out the challenges of longevity, interoperability and barriers to entry in digital music instrument creation, explaining recent developments in ASE and highlighting the possibility that ASE can lower barriers to entry and facilitate creation of interoperable software with greater longevity. Following that, we present case studies wherein we used ASE technology in three distinct ways to develop audio software in the C++ language with the JUCE framework. In case study 1, we re-implement Laurie Spiegel's `Music Mouse' software as a native plugin. In case study 2, we translate Pachet's `Continuator' system from Python into a native plugin. In case study 3, we develop a new 3D user interface for an existing `tracker' sequencer using OpenGL. We describe the experiences of the human developer in the case studies via autoethnographic discussion of the prompt logs and snapshots of the software as it was developed. We identify effective practice for ASE use in this domain and suggest future steps for the work involving evaluation of the method with non-programmer musicians.

### [2] [Correctness-Aware Repository Filtering Under Maximum Effective Context Window Constraints](https://arxiv.org/abs/2605.14362)
*Shweta Mishra*

Main category: cs.SE

TL;DR: 提出了一种正确性感知的上下文卫生框架，通过基于文件大小的启发式过滤（SizeFilter/HybridFilter），在无索引、亚毫秒级开销下实现高比例token压缩，并在代码任务上显著提升准确率、降低幻觉率。


<details>
  <summary>Details</summary>
Motivation: 大语言模型的上下文窗口存在"最大有效上下文窗口"限制，当仓库包含大型非代码构件时，任务相关源代码会被挤出窗口，导致质量下降。现有语义检索方法需构建索引且开销大，亟需一种高效、轻量的上下文过滤方法。

Method: 提出正确性感知的上下文卫生框架：在分词前基于文件大小进行启发式过滤（SizeFilter），仅依赖OS层stat()元数据，开销<0.01 ms/文件；同时设计了HybridFilter混合策略。无需索引构建，不依赖语义检索。

Result: 在10个开源仓库（22,046文件，5种语言）上，SizeFilter（阈值1 MB）实现79.6%平均token压缩，开销仅0.30 ms；HybridFilter达89.3%压缩且方差最低。18个代码任务上，过滤后文件级准确率从25%提升至72%，幻觉率从61%降至17%。

Conclusion: 基于文件大小的启发式过滤是一种极低成本且有效的上下文压缩手段，能够显著提升LLM在代码任务上的表现，为上下文窗口管理提供了轻量级、可复现的实用方案。

Abstract: Context window efficiency is a practical constraint in large language model (LLM)-based developer tools. Paulsen [12] shows that all tested models degrade in accuracy well before their advertised context limits the Maximum Effective Context Window (MECW) which makes context construction a quality problem, not just a cost one. Modern software repositories routinely contain large non-code artifacts compiled datasets, binary model weights, minified JavaScript bundles, and gigabyte-scale log files that overflow the context window and push out task-relevant source code. We present a correctness-aware context hygiene framework: a pre-execution, size-based heuristic filter that intercepts repository scans before tokenization, using only OS-level stat() metadata with sub-millisecond overhead. Semantic retrieval approaches such as RepoCoder, GraphRAG, and AST-based chunking require index construction and query-time inference before any filtering decision is reached. Our framework, by contrast, requires no indexing and operates at <0.01 ms per file decision. Across 10 real open-source repositories (22,046 files, 5 languages), the proposed SizeFilter at θ=1 MB achieves 79.6% (\pm13.2%) mean token reduction at 0.30 ms overhead: the HybridFilter achieves 89.3% (\pm9.0%) the lowest variance of any filter evaluated. A token-density study across 2,688 files confirms a strong linear correlation (Pearson r=0.997, k=0.250 tokens/byte). A limited-scope evaluation (18 tasks, CodeLlama-7B-Instruct) yields 72% file-level accuracy under filtering versus 25% at baseline; hallucination frequency declines from 61% to 17%. All code and data are released for reproducibility.

</details>
